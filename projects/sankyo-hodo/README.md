## 背景・課題
三共舗道は北海道を拠点とする道路舗装会社。親会社あり。施工計画書の作成業務が属人化・非効率化しており、AIエージェントによる自動化を検討中。

**主な課題:**
- 過去の資料を探すのに手間がかかる
- 差し戻しが多い
- ベテランの経験に依存している（40代以降は自分で作れるが若手は難しい）
- 若手が資材数量の計算式を理解していない・内容を理解しないまま作成
- 現場ごとにゼロからExcelを編集する非効率さ
- 単純業務に時間を取られ、若手育成や新技術導入に手が回らない

**定量データ（2/24 MTGで確認）:**
- 年間施工計画書作成: 40件
- 1件あたり作成時間: 約200時間
- 年間合計: 約8,000時間
- AI導入後の見込み: 40時間/件（80%をAIがカバー、残り20%は確認・調整）

## 提案内容

### プロダクト概要
施工計画書作成AIエージェント: 入札図書を入力 → AIが構造化データ作成 → 社内DBから類似工事検索 → 各章を自動作成 → Excel雛形に埋め込んで出力

### ステップ構成
- **ステップ1（3ヶ月）**: 検証・要件定義 - 機能の精度検証、要件の詳細化
  - 費用: 990万円（税込）「先行検証特別パートナープラン」
  - 条件: フィードバック協力、対外PR協力、データ提供
- **ステップ2（3-5ヶ月）**: 構築・運用移行 - ユーザーが光上で利用できる形に構築
  - 費用: 月額（売上の1-9%、実績3-4%。要件に応じて調整可能）
- **運用開始目標**: 10月〜

### 技術構成
- Azure OpenAI GPT-4（章生成・情報抽出）
- Azure AI Search（過去施工計画書の類似検索）
- Azure Document Intelligence（柱状図解析・PDF OCR）
- Google Gemini（施工方法の線画イラスト生成）
- Excel形式出力（openpyxl）

## メモ
- 北海道開発局・北海道建設管理部の2つの発注者に対応する必要あり
- Excel形式での出力が絶対要件（Word不可）
- 開発局用と建設管理部用でExcelテンプレートを分けて保存する想定（先方発言）
- 過去施工計画書のデータ構造化・インデックス化が前提条件
- 図面認識の精度が重要な検証ポイント（先方が光で図面読み込みテスト済み→「100%ではないが素人1-2年生よりは回答が返ってくる」）
- 5年〜10年先を見据えた導入意向（若手育成の観点）
- 工程表（計画工程表）の中身はAIでは難しい→カレンダー出力のみ対応
- インフラゼロカーボ登録等のインターネット利用作業は別途
- 大成建設がAI施工計画書作成の事例あり（競合情報として先方に提示済み）
- 施工計画書以外にも技術提案書・実行予算書等への将来展開構想あり
