## 背景・課題
日本設備工業は建築設備（空調・衛生・消防等）の施工会社。設計から施工まで全工程のDX化を社長主導で推進中。

**主な課題:**
- 施工に必要な書類（施工要領書、安全作業指示書、技術提案書等）が膨大で作成が大変
- 所轄官庁への届出書類の提出漏れリスク（着工遅延の原因に）
- 空調容量計算等の積み上げ計算を毎回手動で実施
- 過去図面の検索・活用が非効率
- Revitの本格運用がまだ（導入途上）
- デジビルに1億円/年投資しているが「全然うまくいっていない」状況

## 3つのワークストリーム

### 1. DX（川分芳子リード）
Revit開発を中心とした設計・施工DX。先方の要望リスト（13項目以上）に対応。
- No.1 負荷計算・機器選定
- No.2 機器自動配置
- No.3 配管・ダクト経路探索
- No.4 見積書・室諸元表・特記など出力
- No.5 施工シミュレーション・施工ステップ図出力
- No.6 設備改修工事計画の自動提案
- No.7 点群 to BIM
- No.9 過去図面検索
- No.13 工程表作成支援
- 予算: ~5,000万円

### 2. デジビル（デジタルビルダー）
既存SaaS。OCR等の機能紹介、Revit連携。書類の提出管理・リマインド。
- 現状1億円/年投資中だが成果不十分
- BOX/SharePoint連携要望あり

### 3. Hikari/光（川越奎拓リード）
AIエージェントによる書類作成支援。
- **施工要領書**: 過去資料ベースで叩き台自動生成
- **安全作業指示書**: 危険作業リストからの自動生成
- **技術提案書**: 空調容量計算等の積み上げ計算支援（初期投資vsランニングコスト比較）
- 予算: ~5,000万円
- BOX連携要望あり

### 3つの連携構想（社長プレゼン向け）
```
Revit（DX） → 図面から必要書類リスト・期限を自動抽出
    ↓
デジビル → 書類の提出管理・リマインド・ルーティング
    ↓
Hikari/光 → 書類の叩き台をAIで自動作成
```
