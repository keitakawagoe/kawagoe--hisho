## 1. 建築設備DX・AIエージェントの基礎知識

### 1-1. 建設DXの全体像

建設業界のDXは、i-Construction 2.0（国土交通省）を軸に、2040年度までに**省人化3割・生産性1.5倍**を目標として推進されている。2025年度からは「建築GX・DX推進事業」が開始され、BIM導入企業に対し設計業務で最大3,500万円、施工業務で最大5,500万円の補助金が交付される。

| 項目 | 内容 |
|------|------|
| 政策目標 | 2040年度までに省人化3割、生産性1.5倍（i-Construction 2.0） |
| BIM確認申請 | 2026年春から一部自治体で開始、2029年に本格導入 |
| 補助金 | 設計業務 最大3,500万円、施工業務 最大5,500万円 |
| DX対策実施率 | わずか14.4%（建設業従事者の6割が「2025年の崖」を未認識） |
| AI活用率 | 建設・不動産業で9.4%（全業種中最低水準、ただし導入企業の約9割が効果を実感） |

### 1-2. AIエージェントとは

AIエージェントは、「施工計画書を作成して」といった大まかな指示を与えるだけで、資料参照・法令調査・工法考案など必要なタスクを**自律的に実行**する。従来のチャットAI（1問1答）とは異なり、複数ステップの業務を自動で遂行する点が特徴。

| 従来のAIチャット | AIエージェント |
|---|---|
| 1問1答 | 複数ステップを自律実行 |
| ユーザーが手順を指示 | 目標を与えると自分で計画 |
| テキスト生成が中心 | ファイル操作・外部連携も可能 |
| 単一タスク | 書類作成→チェック→提出管理まで一気通貫 |

> **商談での仮説**: 日本設備工業は「何をどこに頼めばいいか整理できていない」状態（2/10内部MTG）。AIエージェントが業務の分解・実行を自動化することで、整理能力の不足をシステムで補える。
>
> **社長プレゼンでの示唆**: 社長は技術者 → 「AIが何をするか」の具体的なプロセスフロー（特記仕様書読込→書類リスト生成→叩き台作成）を見せることで技術的信頼を得られる。

## 3. 【テーマ1】施工要領書の業務実態とドメイン知識

### 3-1. 施工計画書・施工要領書・作業手順書の3層構造

建築設備工事の現場では、品質・安全管理のために**3層の書類体系**が存在する。この3つの関係を正確に理解することが、AIエージェント設計の出発点となる。

| 項目 | 施工計画書 | 施工要領書 | 作業手順書 |
|------|-----------|-----------|-----------|
| **作成者** | 元請会社（ゼネコン） | 協力会社（サブコン＝日本設備工業） | 現場管理者・作業員 |
| **提出先** | 発注者（施主） | 元請会社（ゼネコン） | 現場内で活用 |
| **重視する点** | 工事全体の計画・方針 | **品質の確保** | **安全性の確保** |
| **内容の粒度** | 全体的・総合的 | 具体的な施工方法 | より細分化された作業手順 |
| **法的根拠** | 共通仕様書で提出要求 | 建設業法第26条関連 | 労働安全衛生法 |

> **実務上の重要ポイント**: 「施工計画書」と「施工要領書」の用語の使い分けは**会社によって異なる**。ゼネコンAでは「施工計画書」と呼ぶものを、ゼネコンBでは「施工要領書」と呼ぶケースがある。AIエージェント設計時にはこの表記揺れへの対応が必要。

**3層の関係性**:
```
施工計画書（元請：ゼネコンが作成）
  └→ 施工要領書（下請：日本設備工業が作成）← Hikariの主要ターゲット
      └→ 作業手順書（現場：作業員が作成）← 安全作業手順書もここに含まれる
```

### 3-2. 施工要領書の構造と適用図書

#### 施工要領書の定義と法的位置づけ

施工要領書とは、**協力業者（下請け）が元請けの建築会社に提出する書類**のことで、使用材料や施工内容、安全対策などを記載する。元請会社が作成した施工計画書をもとに、協力会社（日本設備工業）が自社の機材や技術レベルに応じて作成し、作業手順や流れを具体化する。

施工要領書を作成した場合、**その通りに施工を進めなければならない**。変更するには多くの人から承認を得る必要があるため、最初の作成段階で詳細な部分まで作り込む必要がある。

#### 適用図書の優先順位（実務上の重要ルール）

施工要領書の「総則」には、適用図書の優先順位が明記される。**原則として現場優先**であり、優先順位は以下の番号順:

| 優先順位 | 適用図書 | 説明 |
|----------|----------|------|
| 1 | **見積要項**（現場説明記録・質疑応答記録含む） | 現場での取り決めが最優先 |
| 2 | **特記仕様書** | 当該工事固有の仕様・条件 |
| 3 | **設計図** | 建築・設備の設計図面 |
| 4 | **共通仕様書**（日本建築家協会監修 建築設備工事共通仕様書20XX年度版） | 業界標準の仕様基準 |

> **AIエージェントへの示唆**: 例えば「設計図にはこう書かれているけど、現場調査したら予想外の配管が埋まっていたので配管ルート変えましょう」という場合、見積要項（現場説明記録）が設計図より優先される。AIが書類を生成する際にも、この優先順位ロジックを組み込む必要がある。

#### 施工要領書の総則構成（実際の目次例）

```
第一章　総則
  1-1. 適用範囲および目的
  1-2. 適用図書（優先順位の明記）
  1-3. 質疑・協議・変更
  1-4. 承認
  1-5. 関係者・専門工事業者への周知
```

日本建設業連合会 関西支部のひな形では、全体の目次構成として以下が標準:
1. 総則 → 2. 工事概要 → 3. 工程表 → 4. 施工管理体制 → 5. 施工要領 → 6. 自主検査 → 7. 安全・衛生管理 → 8. その他

### 3-4. ゼネコン別フォーマットの差異問題

日本設備工業がサブコンとして施工要領書を提出する相手（元請ゼネコン）によって、書類のフォーマット・記載項目・提出ルールが異なる。これが作成負荷の大きな要因となっている。

| 差異の種類 | 具体例 |
|------------|--------|
| **書式・テンプレート** | 竹中工務店は独自Excel書式、大成建設はWord書式、清水建設はExcel+図面添付が基本 |
| **記載項目の粒度** | 同じ配管工事でも、A社は「材料一覧」が必須、B社は「施工写真計画」が必須 |
| **提出タイミング** | 着工前に全工種一括提出 vs 工種ごとに着手前提出 |
| **承認フロー** | 現場所長承認のみ vs 本社技術部の承認が追加で必要 |
| **安全計画の統合** | 施工要領書に安全計画を含める vs 別書類として提出 |

> **AIエージェントへの示唆**: 「汎用的な叩き台を生成し、ゼネコン別の差異は手動で微調整」が桃井氏の要望。AIエージェントでは**ゼネコン名を選択パラメータ**として、テンプレート切替＋記載項目の自動調整を行う設計が有効。

## 3B. 【テーマ2】技術提案書（空調設備）のドメイン知識

### 3B-1. VE提案・設計VEとは

技術提案書の根幹にある概念が**VE（バリューエンジニアリング）**。日本設備工業が作成する技術提案書は、施主やゼネコンに対して「同等以上の機能をより低コストで実現する代替案」を提案するものであり、VE提案の一形態。

| 概念 | 定義 |
|------|------|
| **VE** | Value Engineering。製品やサービスの品質・機能を維持したままコストを削減する手法 |
| **設計VE** | 設計段階でVEを適用。発注者が求める建物の機能とコスト・スケジュールの観点から費用対効果の向上を図る |
| **施工VE** | 施工段階（契約後）にサブコンから提案。施工方法・使用機器の代替提案 |

**設計VEの検討視点**:
- 発注者が求める建物の機能を満たしているか？
- 過剰な仕上げグレードになっていないか？
- 同じ機能を満たす安価な建材・機器はないか？
- ランニングコストを含めたLCC（ライフサイクルコスト）で最適か？

> **日本設備工業の文脈**: 空調方式のA/B/C/D案比較は典型的な**設計VE提案**。設計段階で複数の空調方式を比較し、イニシャルコスト・ランニングコスト・省エネ性能・快適性の総合評価を行う。

### 3B-2. 空調方式の種類と比較

技術提案書では、建物の用途・規模・予算に応じて最適な空調方式を選定・比較する。主要な空調方式:

#### 大分類: 中央熱源方式（セントラル空調） vs 個別熱源方式（個別空調）

| 項目 | セントラル空調 | 個別空調 |
|------|---------------|---------|
| **概要** | 中央機械室に熱源を集約し、冷温水やエアで各室に供給 | 各室・各ゾーンに個別の室外機＋室内機を設置 |
| **代表方式** | 単一ダクト方式、FCU＋外調機、二重ダクト | VRF（ビル用マルチ）、パッケージエアコン |
| **適する建物** | 大規模オフィス、病院、ホテル、工場 | 中小規模ビル、テナントビル、店舗 |
| **イニシャル** | 高い（大型機器・配管・ダクト工事） | 比較的安い（機器単位で導入） |
| **ランニング** | 効率運転で低い傾向（大型機器の高COP） | 部分負荷時に有利（個別制御） |
| **メンテナンス** | 中央機械室で一括管理 | 個別機器ごとの保守が必要 |
| **テナント対応** | 課金が複雑（按分計算） | 個別メーター対応が容易 |

#### 主要な空調方式の詳細

| 方式 | 特徴 | 適用建物 |
|------|------|----------|
| **VRF（Variable Refrigerant Flow）** | ビル用マルチエアコン。冷媒量を可変制御。ダイキン・三菱電機等が主力 | 中小規模オフィス、テナントビル |
| **VAV（Variable Air Volume）** | 送風量を可変制御。各室の温度を個別にコントロール | 大規模オフィス（セントラルの一種） |
| **天井カセット型** | 天井埋込型の室内機。4方向吹出が一般的。意匠性に優れる | オフィス、店舗、ホテル客室 |
| **ダクト接続型** | 室内機を天井裏に設置し、ダクトで送風。見た目すっきり | オフィス、病院、ホテル |
| **FCU＋外調機** | ファンコイルユニット（個別温度制御）＋外調機（換気・除湿）の組合せ | ホテル、病院、大規模オフィス |
| **床吹出空調** | OAフロア下から空調空気を吹出す。サーバールーム等で採用 | データセンター、OAフロアオフィス |

### 3B-3. 空調負荷計算の基礎知識

技術提案書の根拠となるのが**空調負荷計算（熱負荷計算）**。室内をある一定の温湿度に保つために、空気から取り除くべき熱量（冷房負荷）または供給すべき熱量（暖房負荷）を算出する。

#### 熱負荷の構成要素

| 大分類 | 負荷要素 | 内容 |
|--------|----------|------|
| **外皮負荷** | 構造体負荷 | 外壁・屋根を通しての熱貫流量。壁の構成材料・断熱性能で決まる |
| | 内壁負荷 | 間仕切り壁・室内ドア・床・天井からの熱貫流量 |
| | ガラス面負荷 | 窓ガラスを通しての熱貫流量。ガラスの種類・厚さ・Low-E加工で大きく変わる |
| **日射負荷** | 直達日射 | 窓から直接侵入する日射熱。**方位と時刻で大きく変動** |
| | 構造体日射 | 外壁面に当たる日射の影響で熱伝導により侵入する熱 |
| **室内負荷** | 照明発熱 | LED vs 蛍光灯で大きく異なる（LED: 約10W/m2、蛍光灯: 約20W/m2） |
| | 人体発熱 | 成人1人あたり約**120W（顕熱）+ 50W（潜熱）**、活動量で変動 |
| | 機器発熱 | OA機器、サーバー、厨房機器等。データセンターでは1kW/m2超 |
| **外気負荷** | 換気負荷 | 外気を取り入れることによる冷暖房負荷。**全体の30-50%**を占めることも |
| | すきま風 | 建物の気密性能による自然換気負荷 |

> **冷房 vs 暖房の違い**: 冷房負荷は日射熱の影響を受けて**時刻と方位で大きく変動**（南面と北面で最大負荷の発生時刻が異なる）。暖房負荷は日射を考慮しないため変動が小さい。

#### 主な熱負荷計算ツール

| ツール | 提供元 | 特徴 |
|--------|--------|------|
| **NewHASP/ACLD** | 建築設備技術者協会（JABMEE） | 日本の代表的な非定常熱負荷計算プログラム。複数年計算、間欠空調対応。世界気象データにも対応 |
| **HASP/ACSS/8502** | 同上 | 空調システムの期間消費エネルギー計算 |
| **DK-BIM** | ダイキン工業 | Revit連携のクラウド型空調設計支援。機器配置・冷媒選定・騒音計算を自動化。**無料** |
| **STABRO負荷計算** | インフィールド | 最大熱負荷計算ソフト。DK-BIMとの連携で機器選定を効率化 |
| **BEST** | 建築環境・省エネルギー機構 | 建築物総合エネルギーシミュレーションツール。省エネ法適合判定に使用 |
| **EnergyPlus** | 米国DOE | オープンソースのエネルギーシミュレーション。研究用途が中心 |

### 3B-4. イニシャルコスト vs ランニングコスト（LCC分析）

技術提案書のA/B/C/D案比較の核心は**LCC（ライフサイクルコスト）分析**。

**LCC = イニシャルコスト + Σ(年間ランニングコスト) + Σ(メンテナンスコスト) + 廃棄・処分コスト**

| コスト区分 | 内容 | 割合の目安（建物30年） |
|------------|------|----------------------|
| **イニシャルコスト** | 設計費、機器購入費、施工費、試運転調整費 | **約20-25%** |
| **ランニングコスト** | 電気代、ガス代、水道代 | **約50-60%** |
| **メンテナンスコスト** | 定期点検費、フィルター交換、冷媒補充、修繕費 | **約15-25%** |
| **廃棄コスト** | 機器撤去・処分費、フロン回収費 | **約2-5%** |

> **商談での仮説**: 日本設備工業の技術提案書では、空調方式ごとのLCC比較が核心。しかし「毎回手動で計算している」（桃井氏）。AIエージェントで**建物条件→複数方式の自動比較→PowerPointテンプレート出力**を自動化すれば、提案スピードと精度の両方が向上する。

#### 技術提案書のPowerPoint構成（想定）

日本設備工業が作成する技術提案書の典型的な構成:

| スライド | 内容 |
|----------|------|
| 1 | 表紙（工事名・提案者・日付） |
| 2 | 建物概要（用途・面積・階数・構造） |
| 3 | 設計条件（室内温湿度条件、外気条件、在室人員） |
| 4 | 空調方式比較表（A/B/C/D案の概要） |
| 5 | 各案の系統図（空調フロー図） |
| 6 | イニシャルコスト比較（機器費+施工費） |
| 7 | ランニングコスト比較（年間電気代・ガス代） |
| 8 | LCC比較グラフ（10年/20年/30年） |
| 9 | 省エネ性能比較（COP、年間一次エネルギー消費量） |
| 10 | 推奨案と理由 |

> **社長プレゼンでの示唆**: 社長は設備技術者 → 空調容量計算の具体的な数値（kW/m2、COP、消費電力）を1-2例示すことで技術的信頼性を確保（3/4 MTGでの川分氏提案）。

## 3C. 【テーマ3】安全作業手順書のドメイン知識

### 3C-1. 安全作業手順書の法的根拠

#### 労働安全衛生法の構造

| 法令 | 位置づけ |
|------|----------|
| **労働安全衛生法**（昭和47年法律第57号） | 基本法。職場での安全と健康を守り、快適な作業環境の形成促進を目的 |
| **労働安全衛生法施行令**（政令） | 法の細部を規定 |
| **労働安全衛生規則**（安衛則、省令） | 具体的な行動指針。実務上最も参照される |

#### 作業手順書に関連する法的義務

事故・災害の原因としての管理的欠陥の中で、**最も多く原因に挙げられるのが「作業手順の誤り」**。作業手順は作業員任せではなく、事業者が積極的に関与決定すべき事柄。

| 法的根拠 | 内容 |
|----------|------|
| **安衛法第30条** | 特定元方事業者の義務。関係請負人の安全管理の統括 |
| **元方事業者による建設現場安全管理指針** | 「元方事業者は、関係請負人に対し、労働災害防止に配慮した作業手順書を作成するよう指導すること」 |
| **安衛則** | 作業手順書の作成は**一次事業者（日本設備工業）が会社の責任で作成**し、元請事業者に内容の確認を受ける |

> **実務上の重要ポイント**: 作業手順書は原則として**一次事業者（サブコン）が会社の責任で作成**し、元請事業者に内容の確認をしてもらう。日本設備工業の場合、自社が一次事業者として安全作業手順書を作成し、元請ゼネコンに提出・承認を受ける流れ。

### 3C-4. KY活動（危険予知活動）との関連

安全作業手順書は**KY活動（危険予知活動）**の基礎資料として使われる。KY活動は毎朝の作業開始前に実施される。

#### 基礎4ラウンド法（KYT4ラウンド法）

| ラウンド | テーマ | 内容 |
|----------|--------|------|
| 第1R | **現状把握**「どんな危険が潜んでいるか」 | 作業環境・内容・使用機器を観察し、危険箇所を多数洗い出す |
| 第2R | **本質追求**「これが危険のポイントだ」 | 重要度の高い危険を絞り込み、危険のポイントを特定 |
| 第3R | **対策樹立**「あなたならどうする」 | 危険に対する対策案を多く出す |
| 第4R | **目標設定**「私たちはこうする」 | すぐ実施すべき対策から行動目標を作成。「○○をする」という肯定形で |

> **AIエージェントの付加価値**: 安全作業手順書の「危険有害要因」「対策」欄の内容は、そのままKY活動の第1R・第3Rのネタになる。AIが過去のヒヤリハット事例DBも参照して、**作業手順書とKYシートを同時生成**できれば、現場の毎朝のKY活動も効率化できる。

## 5. 建築設備業界の動向と日本設備工業の位置づけ

### 5-1. 建設業界の重層下請構造とサブコンの位置づけ

日本の建設業は**重層下請（ピラミッド）構造**になっている。日本設備工業は「サブコン」として、このピラミッドの中間層に位置する。

```
発注者（施主: デベロッパー、官公庁等）
  └→ 元請（ゼネコン: 竹中工務店、大成建設、清水建設等）
      └→ 一次下請（サブコン: 日本設備工業） ← ここ
          └→ 二次下請（職人集団、専門工事会社）
              └→ 三次下請（個人事業主、一人親方）
```

#### サブコンの4つの専門分野

| 分野 | 主な工事内容 | 代表企業 |
|------|-------------|----------|
| **電気設備** | 受変電設備、照明、通信、防災 | きんでん、関電工、九電工 |
| **空調設備** | 空調、換気、冷暖房 | 高砂熱学、大気社、三機工業 |
| **衛生設備** | 給排水、給湯、ガス、消防 | 三機工業、ダイダン |
| **総合設備** | 電気+空調+衛生の複合 | ダイダン、新日本空調 |

日本設備工業は**空調＋衛生設備**を専門とするサブコン。親会社の高砂熱学工業グループに属する。

#### 発注方式の種類（日本設備工業の受注パターン）

| 発注方式 | 内容 | 日本設備工業への影響 |
|----------|------|---------------------|
| **一式請負** | ゼネコンが全体を受注し、設備工事をサブコンに下請発注 | 最も一般的。ゼネコンのフォーマットに従う義務 |
| **分離発注** | 発注者が設備工事を直接サブコンに発注 | 自社フォーマットで書類作成可能。自由度が高い |
| **コストオン契約** | 施主がサブコンを指定し、ゼネコンに管理費を上乗せ | 技術力で指名されるため、技術提案書の品質が受注に直結 |

> **商談での活用**: 日本設備工業は「一式請負」でゼネコンの下請として入ることが多い → ゼネコンごとに施工要領書のフォーマットが異なる根本原因。「分離発注」や「コストオン」では自社フォーマットが使えるため、書類作成の負荷が下がる。

### 5-6. 空調負荷計算のAI自動化

日本設備工業が技術提案で求めている「積み上げ計算」の効率化に関連する技術動向:

| ツール/サービス | 提供企業 | 機能 |
|----------------|----------|------|
| DK-BIM | ダイキン工業 | Revit連携で熱負荷計算→機器選定→自動配置（無料） |
| DK-CONNECT | ダイキン工業 | AI熱負荷予測による遠隔自動省エネ制御（2025年春リリース） |
| ConsciousAir | アラヤ | 空調最適化AI、冷房期消費電力12%削減実証 |
| AI空調制御クラウド | NTTドコモ | 深層強化学習による消費エネルギー35%以上削減 |
| HUCAST | NTTデータ | AI空調最適化、消費エネルギー最大50%削減 |
| AI設計部長 | 大成建設 | 希望条件から最適設計案を短時間生成、Revit出力 |

**空調容量計算のAI化フロー（想定）**:
1. 建物条件（面積・用途・方位・断熱性能）を入力
2. AIが外気条件・内部発熱を積み上げ計算
3. 複数の空調方式（個別空調/セントラル/VRF等）でA/B/C/D案を自動生成
4. イニシャルコスト vs ランニングコストの比較表を出力
5. PowerPointテンプレートに自動挿入

> **商談での仮説**: ダイキンのDK-BIMは無料だが、日本設備工業の実務フローに完全に合致しているわけではない。Hikariで「過去の技術提案書+計算ロジック」を学習させ、自社仕様の提案書を自動生成する方が実用的。
>
> **社長プレゼンでの示唆**: 社長は設備技術者 → 空調容量計算の具体的な数値（kW/m2、COP、消費電力）を1-2例示すことで技術的信頼性を確保（3/4 MTGでの川分氏提案）。
