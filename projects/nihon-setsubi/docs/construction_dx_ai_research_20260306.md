## 1. 建築設備DX・AIエージェントの基礎知識

### 1-1. 建設DXの全体像

建設業界のDXは、i-Construction 2.0（国土交通省）を軸に、2040年度までに**省人化3割・生産性1.5倍**を目標として推進されている。2025年度からは「建築GX・DX推進事業」が開始され、BIM導入企業に対し設計業務で最大3,500万円、施工業務で最大5,500万円の補助金が交付される。

| 項目 | 内容 |
|------|------|
| 政策目標 | 2040年度までに省人化3割、生産性1.5倍（i-Construction 2.0） |
| BIM確認申請 | 2026年春から一部自治体で開始、2029年に本格導入 |
| 補助金 | 設計業務 最大3,500万円、施工業務 最大5,500万円 |
| DX対策実施率 | わずか14.4%（建設業従事者の6割が「2025年の崖」を未認識） |
| AI活用率 | 建設・不動産業で9.4%（全業種中最低水準、ただし導入企業の約9割が効果を実感） |

### 1-2. AIエージェントとは

AIエージェントは、「施工計画書を作成して」といった大まかな指示を与えるだけで、資料参照・法令調査・工法考案など必要なタスクを**自律的に実行**する。従来のチャットAI（1問1答）とは異なり、複数ステップの業務を自動で遂行する点が特徴。

| 従来のAIチャット | AIエージェント |
|---|---|
| 1問1答 | 複数ステップを自律実行 |
| ユーザーが手順を指示 | 目標を与えると自分で計画 |
| テキスト生成が中心 | ファイル操作・外部連携も可能 |
| 単一タスク | 書類作成→チェック→提出管理まで一気通貫 |

> **商談での仮説**: 日本設備工業は「何をどこに頼めばいいか整理できていない」状態（2/10内部MTG）。AIエージェントが業務の分解・実行を自動化することで、整理能力の不足をシステムで補える。
>
> **社長プレゼンでの示唆**: 社長は技術者 → 「AIが何をするか」の具体的なプロセスフロー（特記仕様書読込→書類リスト生成→叩き台作成）を見せることで技術的信頼を得られる。

### 1-3. 建設業×生成AIの活用領域

| 活用領域 | 具体例 | 成熟度 |
|----------|--------|--------|
| 書類作成支援 | 施工計画書、安全書類、日報の叩き台生成 | 実用段階 |
| ナレッジ検索 | 過去の施工要領書・危険事例をRAGで即座検索 | 実用段階 |
| 図面×AI | BIMモデルから数量拾い、積算、配管経路最適化 | 開発中 |
| 積算自動化 | 図面認識AIによる内装工事積算 | 実用開始 |
| 工程管理 | BIMデータ連携によるスケジュール最適化 | 開発中 |
| 空調設計 | 負荷計算・機器選定の自動化（DK-BIM等） | 一部実用 |

### 2-3. Hikari（AIエージェント）の活用ポイント

桃井氏ヒアリング（3/4 MTG）で得られた具体的な要件:

**施工要領書**:
- 工種別に約**15種類**
- ゼネコンごとにフォーマットが異なる → 汎用的な叩き台を生成し、手動で微調整
- Box内の過去資料を参照して類似文書を自動検索・生成

**安全作業手順書**:
- 重点危険作業 **7項目** + 部長許可部の2分類
- 高所作業 / 地上作業 → 脚立式作業台 / 天井内作業 / ローリングタワー / 足場組立解体... と枝分かれ
- 最末端の階層で類似パターンを検索 → 叩き台生成 → 手修正
- Excelフォーマット（作業名・作業手順・危険有害性・対策・イラスト）

**技術提案書**:
- PowerPointテンプレートに基づく提案書自動生成
- 空調方式のA/B/C/D案比較（イニシャルコスト vs ランニングコスト）
- 概算金額・工事日数の自動算出

> **商談での活用**: 桃井氏は「100%の資料はできなくていい、70%の叩き台でよい」と明言。また「光のエージェントとして左側に並ぶイメージ」と具体的なUI要望あり。
>
> **社長プレゼンでの示唆**: 桃井氏から「課題解決ではなく課題達成の視点で提案すべき」「新しいやり方を強めに、技術的な部分を見せると刺さる」との助言あり。

## 3. 施工書類作成の実務とAI化

### 3-1. 施工要領書の構造

施工要領書は、工事の種類（空調・衛生・消防等）ごとに作成され、施工手順・使用材料・品質基準・安全対策を記載する。日本設備工業の場合:

| 要素 | 内容 |
|------|------|
| 種類数 | 工種別に約15種類 |
| フォーマット | ゼネコンごとに異なる（竹中・大成・清水等で書式が違う） |
| 作成頻度 | 現場ごとに毎回作成（過去のものを流用・修正） |
| 参照データ | 過去の施工要領書（Box保存）、特記仕様書、図面 |
| 作成時間 | 1人あたり勤務時間の約1/3が書類作成に費やされている |

### 3-2. 安全作業手順書のロジックツリー

桃井氏が3/4 MTGで共有した安全作業手順書の分類ロジック:

```
安全作業手順書
├── 重点危険作業（7項目）
│   ├── 高所作業
│   │   ├── 脚立式作業台
│   │   ├── 天井内作業
│   │   ├── 高所作業車使用
│   │   ├── ローリングタワー
│   │   ├── 足場組立解体
│   │   └── 屋上・開口部・端部作業
│   ├── 機器搬出入（クレーン等）
│   ├── 既設撤去解体
│   │   ├── タンク / 空調機 / パッケージ / 送風機
│   │   └── ...
│   ├── 火気使用設備（溶接・ロウ付等）
│   ├── 消防設備
│   ├── コンクリート・掘削・水抜き
│   └── 電気関係（電気室・サーバールーム）
├── 部長許可部
└── その他危険作業
    ├── 第三者関連作業
    ├── 回転工具使用（丸ノコ・サンダー等）
    └── ...
```

### 3-3. AIによる書類作成のアプローチ

| ステップ | 内容 | 技術要素 |
|----------|------|----------|
| Step 1 | ユーザーが工種・ゼネコン・作業内容を選択 | UIフロー設計 |
| Step 2 | Box内の過去書類から類似文書を検索 | Box API連携 + RAG |
| Step 3 | 類似文書をベースに叩き台を自動生成 | 生成AI（LLM） |
| Step 4 | Excelフォーマットで出力・ダウンロード | テンプレートエンジン |
| Step 5 | ユーザーが手修正・提出 | ブラウザ上での編集（将来） |

**将来構想（特記仕様書→全書類自動生成）**:
1. 特記仕様書PDFをアップロード
2. AIが内容を解析し、必要な書類リストを自動生成
3. 各書類の叩き台を一括作成
4. Revitの時系列データと連携し、提出期限を自動設定

> **商談での仮説**: 桃井氏は「特記仕様書1つ入れたら全書類が出てくるのが理想」と述べつつ、「段階的にステップを踏んで進めるのが現実的」と理解を示している。社長プレゼンでは理想像を見せつつ、初期フェーズは選択式UIで確実に動くものを提案する。

### 3-4. 大手ゼネコンの先行事例

| 企業 | 取り組み | 技術 |
|------|----------|------|
| 大成建設 | 全体施工計画書作成支援システム | VLM（視覚言語モデル）ベースのマルチモーダル生成AI |
| 清水建設 | 「Lightblue Assistant」による施工要領書検索 | RAG技術で約1,000ページの施工要領書を即座に検索 |
| 竹中工務店 | 「デジタル棟梁」ナレッジ検索システム | 生成AIで社内文書を検索し専門知識を引き出す |
| 北野建設×日立ソリューションズ | 書類作成・技術調査の省力化 | 生成AI技術、2025年春に販売開始 |

> **社長プレゼンでの示唆**: 大手ゼネコンが数億円規模で取り組んでいる施工書類AI化を、日本設備工業は燈のHikariプラットフォーム上で**短期間・低コスト**で実現できるというポジショニング。

### 5-4. 空調負荷計算のAI自動化

日本設備工業が技術提案で求めている「積み上げ計算」の効率化に関連する技術動向:

| ツール/サービス | 提供企業 | 機能 |
|----------------|----------|------|
| DK-BIM | ダイキン工業 | Revit連携で熱負荷計算→機器選定→自動配置（無料） |
| DK-CONNECT | ダイキン工業 | AI熱負荷予測による遠隔自動省エネ制御（2025年春リリース） |
| ConsciousAir | アラヤ | 空調最適化AI、冷房期消費電力12%削減実証 |
| AI空調制御クラウド | NTTドコモ | 深層強化学習による消費エネルギー35%以上削減 |
| HUCAST | NTTデータ | AI空調最適化、消費エネルギー最大50%削減 |
| AI設計部長 | 大成建設 | 希望条件から最適設計案を短時間生成、Revit出力 |

**空調容量計算のAI化フロー（想定）**:
1. 建物条件（面積・用途・方位・断熱性能）を入力
2. AIが外気条件・内部発熱を積み上げ計算
3. 複数の空調方式（個別空調/セントラル/VRF等）でA/B/C/D案を自動生成
4. イニシャルコスト vs ランニングコストの比較表を出力
5. PowerPointテンプレートに自動挿入

> **商談での仮説**: ダイキンのDK-BIMは無料だが、日本設備工業の実務フローに完全に合致しているわけではない。Hikariで「過去の技術提案書+計算ロジック」を学習させ、自社仕様の提案書を自動生成する方が実用的。
>
> **社長プレゼンでの示唆**: 社長は設備技術者 → 空調容量計算の具体的な数値（kW/m2、COP、消費電力）を1-2例示すことで技術的信頼性を確保（3/4 MTGでの川分氏提案）。

### A. 業務課題の仮説
