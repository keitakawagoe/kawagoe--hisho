## HSコード判定の課題（提案の核）

### 現状の問題
- Hakobi SaaSのチャットモードでHSコード判定を試みているが**精度に課題**
- 通関2部（島本）が実際にテストした結果：
  - 柚子100%果汁 → 最初「2009.89-000」と回答（柑橘類なので2009.31/39が正解）→ 指摘後に修正
  - 海鮮チヂミ → 候補は出せるが最適解の特定が困難
- No.1「分類RAG」で勉強会資料をアップロードして判定させているが、ハルシネーションが発生

### Notionの検証プロンプト（No.1 v2）
- 「資料データのみを絶対的な知識源とする厳格な通関士AI」として設計
- 3ステップ判定プロセス：関連ページ検索→基準照合→論理的結論
- テスト商品：レーズン入りバターロール、ビーフカレー、チョコ＆ビスケット、アサイードリンク

### 提案の方向性
- **カスタマイズAIエージェント**: 税関資料・勉強会資料・分類事例をRAGで構造化し、HSコード判定精度を大幅向上
- 既存のHakobi SaaSでは対応しきれない精度要件に対して、専用エージェントで解決
