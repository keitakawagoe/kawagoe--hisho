## 3. 道路舗装工事のドメイン知識

### 3-1. アスファルト舗装の層構造

道路舗装は下から順に5層で構成され、各層の材料・厚さが工事費の大部分を占める。

| 層 | 厚さ目安 | 役割 | 使用材料 |
|---|---|---|---|
| **路床** | 約1m | 舗装と一体で交通荷重を支持し路体へ分散 | 現地盤の土（必要に応じて改良） |
| **下層路盤** | 数十cm | 荷重を分散して路床へ伝達 | クラッシャラン、鉄鋼スラグ、砂 |
| **上層路盤** | 数十cm | 同上（より高品質） | 粒度調整砕石、瀝青安定処理等 |
| **基層** | 4〜6cm | 路盤の不陸を整正し表層荷重を均一伝達 | 加熱As混合物（粗粒度） |
| **表層** | 3〜5cm | 走行面として摩耗・流動・ひび割れに抵抗 | 加熱As混合物（密粒度） |

> **PJとの関連**: 実行予算書の「工種」はこの層構造に対応する（表層工、基層工、路盤工 等）。AIエージェントが工種名から適切な類似工事を検索するには、この階層構造の理解が前提となる。

### 3-2. 主要な舗装工法

| 工法 | 特徴 | 主な用途 |
|---|---|---|
| **密粒度As舗装** | 最も一般的。空隙率が小さく防水性が高い | 一般道路全般 |
| **排水性舗装** | 表層に空隙を持たせ雨水を内部排水 | 高速道路・幹線道路 |
| **透水性舗装** | 雨水を地中へ浸透させる | 歩道・公園・駐車場 |
| **半たわみ性舗装** | 開粒度Asにセメントミルクを浸透 | バス停、交差点、工場構内 |
| **SMA（砕石マスチック）** | 骨材間隙にモルタル充填、耐久性高い | 寒冷地 |

日本の道路舗装は**アスファルト舗装が約95%**を占める（コンクリート舗装は約5%）。さらに**再生合材が全体の7割強**であり、リサイクルが進んでいる。

### 3-3. 公共工事の積算の仕組み

#### 工事費の構成（国交省の積算基準に基づく）

```
工事価格
├─ 直接工事費（材料費 + 労務費 + 機械経費）
├─ 共通仮設費（= 直接工事費 × 率 + 積み上げ分）
│    → 純工事費 = 直接工事費 + 共通仮設費
├─ 現場管理費（= 純工事費 × 率 × 補正係数）  ※概ね5〜8%
│    → 工事原価 = 純工事費 + 現場管理費
├─ 一般管理費等（= 工事原価 × 率）  ※概ね8〜10%（上限23.57%）
│    → 工事価格 = 工事原価 + 一般管理費等
└─ 消費税
```

> **PJとの関連**: 本PJのAIエージェントが「直接工事費の内訳」と「間接費（共通仮設費・現場経費）の内訳」を別々のDBで管理しているのは、この積算構造に対応している。

#### 歩掛かり（ぶがかり）

歩掛とは、**ある作業を行うときに作業員・機械・資材がどれだけ必要かを数量で表したデータ**。国交省が全国の施工実態調査に基づき設定する。

| 項目 | 内容 |
|------|------|
| 計算例 | 人力土砂積込み：土砂1m³あたり普通作業員0.14人 |
| 人工の計算 | `人工 = (人数 × 必要作業時間) / 8時間` |
| 労務費算出 | `労務費 = 所要人数(設計作業量 × 歩掛) × 労務単価` |
| 注意点 | 標準的な施工条件を想定。狭隘地・交通規制等の補正は経験が必要 |

> **商談での活用**: 歩掛かりの理解は若手にとって最大の壁。ベテランは「頭の中に入っている」が、若手は標準歩掛を参照しつつ現場条件への当てはめに苦労する。AIが過去の実績歩掛を提示し、課長がレビューするワークフローが理想。

#### 設計金額・予定価格・実行予算の関係

| 用語 | 作成者 | 目的 |
|---|---|---|
| **設計金額** | 発注者（官公庁） | 積算基準に基づく工事費総額 |
| **予定価格** | 発注者（官公庁） | 入札の上限金額 |
| **見積** | 受注者（施工業者） | 積算に利益を乗せた提示金額 |
| **実行予算** | 受注者（施工業者） | 社内の原価管理目標（**内部資料**） |

> **PJとの関連**: 本PJは「実行予算書」を元データとしているが、最終的な目的は「見積書」の作成支援。実行予算は社内の実勢価格ベースで作成されるため、見積もりの精度向上に直結する。

### 3-6. 道路舗装工事の現場フロー

```
入札情報確認 → 設計図書入手・積算 → 入札（価格or総合評価）
→ 落札 → 契約 → 実行予算作成 → 施工計画書作成
→ 着工（路床→路盤→基層→表層）→ 品質管理試験
→ 完工 → 完成検査 → 引渡し → 請負代金受領
```

| 入札方式 | 特徴 |
|---|---|
| **一般競争入札** | 誰でも参加可能。最低価格者が落札。最低制限価格あり |
| **指名競争入札** | 発注者が信頼できる業者を指名 |
| **総合評価落札方式** | 価格+技術力を総合評価。談合防止にも有効 |

> **PJとの関連**: 見積書の作成は「入札」フェーズ、実行予算書の作成は「受注後」フェーズ。本PJは両方をカバーする。

### 3-7. 原価管理の流れ

```
実行予算作成 → 施工中の月次予実比較 → 差異分析 → 対策実施 → 工事完了後の振り返り
```

実行予算の4大費目：**材料費・労務費・外注費・経費**

> **将来の発展可能性**: 現在のPJは「予算作成」の支援だが、施工後の実績データをフィードバックすれば原価管理の精度向上にもつながる。2/26の会議で塚本部長が「作ったAIベースに、いざ現場やってみてどうだったのっていうフィードバックも戻せるといいな」と発言。

### 5-7. 若手技術者が苦労するポイント（AIで解決すべき課題）

| 課題領域 | 詳細 | AIでの解決方向 |
|---|---|---|
| **歩掛かりの理解** | 表の読み方だけでなく現場条件への補正が必要。暗黙知の要素が大きい | 過去実績から類似条件の歩掛を自動検索 |
| **現場条件の反映** | 標準歩掛は「標準的な施工条件」想定。実際の現場への当てはめは経験が不可欠 | 類似条件の過去工事を提示し、判断材料を提供 |
| **間接費の率計算** | 指数関数を使った率計算は独特。建設業会計特有の勘定科目も複雑 | 過去工事の間接費実績を一覧表示 |
| **実行予算と見積原価の違い** | 見積は限られた時間・情報での最適値。受注後に実行予算で精査し直す必要がある | 過去の実行予算データから精度の高い参考値を提示 |
| **ベテランの暗黙知** | 長年の勘やノウハウは細分化・可視化が困難 | データとして蓄積・検索可能にする |

> **商談での活用**: これらの課題は1/30会議で「頭を使うのは部掛かり」「単価よりも部掛かりの方が知りたい」と具体的に言語化されている。2/26会議では「属人化解消」「精度向上」が目的として確認済み。
